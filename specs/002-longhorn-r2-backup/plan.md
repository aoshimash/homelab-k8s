# Implementation Plan: Introduce Longhorn with R2 Backups

## Summary

Install and manage Longhorn as the persistent storage provider via Flux CD (GitOps), tuned for a single-node cluster (default replica count = 1). Configure Longhorn backups to Cloudflare R2 (S3-compatible object storage) with daily schedules and 30-day retention. Store all backup credentials encrypted with SOPS + age and configure Flux Kustomize decryption during reconciliation.

## Phase Plan

### Phase 0 — Research (output: `research.md`)

Confirm the exact Longhorn chart values and settings required for:
- Single-node tuning (default replica count = 1; avoid degraded-by-design).
- Cloudflare R2 as S3-compatible backup target (endpoint, bucket, credentials, TLS expectations).
- Flux SOPS decryption wiring (Kustomization `spec.decryption.provider=sops` and `secretRef`).

### Phase 1 — Design (outputs: `data-model.md`, `contracts/*`, `quickstart.md`)

- Formalize the entities and invariants around volumes, backups, restore, and credentials.
- Define a reconciliation contract that makes PR review and validation repeatable.
- Provide an operator quickstart for configuring encrypted credentials and verifying backup/restore.

### Phase 2 — Implementation Outline (not tasks; tasks generated by `/speckit.tasks`)

High-level change set:
- Add `k8s/infrastructure/longhorn/` with Flux `HelmRepository` + `HelmRelease`.
- Add an SOPS-encrypted Secret for R2 credentials and any required Longhorn backup target configuration.
- Extend `.sops.yaml` rules to cover `k8s/**/*.sops.yaml` (keep Talos rules intact).
- Enable Flux Kustomization SOPS decryption for the `infrastructure` reconciliation.
- Validate provisioning + backup + restore workflows and document the verification steps.
