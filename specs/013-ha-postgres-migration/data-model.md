# Data Model: Home Assistant SQLite → PostgreSQL Migration

**Feature**: 013-ha-postgres-migration
**Date**: 2026-01-09

## Kubernetes Resources

### 1. Home Assistant Application (`k8s/apps/home-assistant/`)

#### Namespace (existing)

| Field | Value | Notes |
|-------|-------|------|
| name | home-assistant | Existing namespace for Home Assistant |

#### PVC (existing)

| Field | Value | Notes |
|-------|-------|------|
| name | home-assistant-config | Mounted at `/config` |
| size | 5Gi | Longhorn volume |
| contains | `configuration.yaml`, `home-assistant_v2.db`, etc. | SQLite DB lives here pre-migration |

#### Secret: Database URL (new, SOPS-encrypted)

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: home-assistant-db-url
  namespace: home-assistant
type: Opaque
stringData:
  DB_URL: postgresql://<user>:<password>@postgres-cluster-rw.postgres.svc.cluster.local:5432/<database>
```

| Field | Key | Notes |
|-------|-----|------|
| stringData | DB_URL | Injected into Home Assistant as env var |

#### Home Assistant Configuration (tracked file; mounted into pod)

The configuration contains the Recorder settings and references `DB_URL` via `!env_var` so credentials are not committed in plaintext.

```yaml
recorder:
  db_url: !env_var DB_URL
  db_max_retries: 10
  db_retry_wait: 30
```

| Item | Value | Notes |
|------|-------|------|
| db_url source | environment variable | Keeps credentials out of Git |
| retry behavior | max retries + wait | Supports “DB down → HA usable; history may pause” expectation |

#### Deployment (existing; updated)

Key change is injecting the `DB_URL` environment variable from the Secret.

| Field | Value | Notes |
|-------|-------|------|
| env.DB_URL | from Secret `home-assistant-db-url` | Used by `!env_var` |
| volumeMounts | `/config` PVC | SQLite DB source for migration job |

### 2. PostgreSQL Cluster (`k8s/configs/postgres/`)

#### Cluster (existing)

| Field | Value | Notes |
|-------|-------|------|
| name | postgres-cluster | CloudNativePG Cluster |
| namespace | postgres | Dedicated namespace |
| version | 16 | `ghcr.io/cloudnative-pg/postgresql:16` |

#### Service Endpoints (auto-generated by CloudNativePG)

| Endpoint | DNS Name | Port |
|----------|----------|------|
| Read-Write | postgres-cluster-rw.postgres.svc.cluster.local | 5432 |
| Read-Only | postgres-cluster-r.postgres.svc.cluster.local | 5432 |

### 3. One-time Migration Job (short-lived; applied via Git during cutover)

#### Job: `home-assistant-db-migrate` (new; temporary)

Purpose: copy data from the SQLite database file in the Home Assistant PVC into PostgreSQL.

Key properties:

| Field | Value | Notes |
|-------|-------|------|
| runs when | Home Assistant is scaled down | Avoid concurrent writes |
| input | `/config/home-assistant_v2.db` | From PVC |
| output | PostgreSQL DB | Target via `DB_URL` |
| lifetime | short-lived | Remove manifests after success |

## State Transitions (Migration Run)

```
Prepared → Cutover (HA stopped) → Migrating → Verified → Observing (24h) → Completed
                       │              │
                       └──────────────┴──────────────▶ Rollback
```

| State | Description |
|-------|-------------|
| Prepared | DB/user/secret/config ready; backups taken |
| Cutover | Home Assistant stopped to prevent writes |
| Migrating | pgloader Job copies SQLite → PostgreSQL |
| Verified | UI and basic checks confirm history continuity |
| Observing | 24h stability window; rollback available |
| Completed | Migration considered final; temporary resources cleaned up |
| Rollback | Restore HA to SQLite and/or restore from backup |
