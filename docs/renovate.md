# Renovate - Automated Dependency Updates

This document describes the Renovate configuration and operational procedures for the homelab-k8s repository.

## Overview

[Renovate](https://github.com/renovatebot/renovate) is a dependency update tool that automatically creates pull requests when new versions of dependencies are available.

## What Renovate Manages

### Automatically Tracked

| Type | Location | Manager |
|------|----------|---------|
| Helm Charts | `k8s/infrastructure/*/helmrelease.yaml` | flux |
| GitHub Actions | `.github/workflows/*.yaml` | github-actions |

### Components (6 HelmReleases)

- Cilium (CNI)
- Longhorn (Storage)
- Tailscale Operator
- Grafana Alloy
- kube-state-metrics
- metrics-server

### Not Tracked (Manual Upgrade Required)

| Component | Location | Reason |
|-----------|----------|--------|
| Talos Linux | `infra/talos/talconfig.yaml` | Core OS, requires careful upgrade |
| Kubernetes | `infra/talos/talconfig.yaml` | Managed by Talos |
| Flux CD | `k8s/flux/flux-system/gotk-components.yaml` | Auto-generated by Flux CLI |

## Configuration

The Renovate configuration is stored in `renovate.json5` at the repository root.

### Key Settings

| Setting | Value | Description |
|---------|-------|-------------|
| `prConcurrentLimit` | 10 | Maximum concurrent open PRs |
| `separateMajorMinor` | true | Separate PRs for major version updates |
| `automerge` | false | All PRs require manual review |
| `labels` | `dependencies`, `renovate` | Labels applied to PRs |

### Excluded Paths

The following paths are excluded from Renovate scanning:

- `**/*.sops.yaml` - SOPS encrypted secrets
- `.sops.yaml` - SOPS configuration
- `k8s/flux/flux-system/gotk-components.yaml` - Flux auto-generated

## Workflow

### When a New Version is Available

1. Renovate detects the new version (within 24 hours)
2. A PR is automatically created with:
   - Version change details
   - Release notes link
   - `dependencies` and `renovate` labels
3. Review the PR and test if needed
4. Merge the PR
5. Flux automatically applies the change to the cluster

### Reviewing PRs

Before merging a Renovate PR:

1. Check the release notes for breaking changes
2. For major version updates, verify compatibility
3. Consider testing in a staging environment for critical components

## Manual Upgrade Procedures

### Talos Linux Upgrade

Talos and Kubernetes versions are defined in `infra/talos/talconfig.yaml`:

```yaml
talosVersion: v1.12.0
kubernetesVersion: v1.35.0
```

To upgrade:

1. Check [Talos Release Notes](https://www.talos.dev/latest/introduction/support-matrix/)
2. Update versions in `talconfig.yaml`
3. Regenerate configs: `talhelper genconfig`
4. Apply: `talosctl upgrade --image <new-image>`
5. Commit changes to Git

### Flux CD Upgrade

Flux components are in `k8s/flux/flux-system/gotk-components.yaml`:

1. Check [Flux Release Notes](https://github.com/fluxcd/flux2/releases)
2. Run: `flux install --export > k8s/flux/flux-system/gotk-components.yaml`
3. Commit and push changes
4. Flux will self-upgrade

## Troubleshooting

### PRs Not Being Created

1. Check the [Renovate Dashboard](https://developer.mend.io/github/aoshimash/homelab-k8s) for status
2. Verify `renovate.json5` syntax is valid
3. Check if the component is in `ignorePaths`

### Too Many PRs

Reduce `prConcurrentLimit` in `renovate.json5` or close stale PRs.

### Dependency Not Detected

Verify the file path matches the `fileMatch` pattern for the relevant manager.
