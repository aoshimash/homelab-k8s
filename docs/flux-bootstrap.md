# Flux Bootstrap Guide

This document describes how to bootstrap Flux CD for GitOps management of the homelab Kubernetes cluster.

## Prerequisites

### Required Tools

Install the Flux CLI:

```bash
brew install fluxcd/tap/flux
```

Verify the installation:

```bash
flux --version
```

### Cluster Requirements

- A running Kubernetes cluster (see [Talos Bootstrap Guide](./talos-bootstrap.md))
- `kubectl` configured to access the cluster
- Cluster admin permissions

Verify cluster access:

```bash
kubectl cluster-info
flux check --pre
```

## GitHub Authentication

### Create a Fine-grained Personal Access Token (PAT)

For security, we use a fine-grained PAT which allows repository-specific permissions.

1. Go to GitHub: **Settings** → **Developer settings** → **Personal access tokens** → **Fine-grained tokens**

2. Click **Generate new token**

3. Configure the token:

   | Setting | Value |
   |---------|-------|
   | Token name | `flux-homelab-k8s` |
   | Expiration | No expiration (or your preference) |
   | Resource owner | Your account (`aoshimash`) |
   | Repository access | **Only select repositories** → `homelab-k8s` |

4. Set **Repository permissions**:

   | Permission | Access Level |
   |------------|--------------|
   | Administration | Read-only |
   | Contents | Read and write |
   | Metadata | Read-only |

   > **Note**: If you want to use SSH deploy keys instead of token auth, set `Administration` to `Read and write`.

5. Click **Generate token** and copy the token

6. Export the token as an environment variable:

   ```bash
   export GITHUB_TOKEN=<your-fine-grained-pat>
   ```

## Project Structure

After bootstrap, the repository will have the following structure:

```
homelab-k8s/
├── docs/
│   ├── talos-bootstrap.md
│   └── flux-bootstrap.md
├── infra/
│   └── talos/                     # Talos configuration (outside Flux)
├── k8s/                           # Flux-managed Kubernetes resources
│   ├── flux/                      # Flux system (auto-generated by bootstrap)
│   │   └── flux-system/
│   │       ├── gotk-components.yaml
│   │       ├── gotk-sync.yaml
│   │       └── kustomization.yaml
│   ├── infrastructure/            # Cluster infrastructure (ingress, cert-manager, etc.)
│   │   └── kustomization.yaml
│   └── apps/                      # User applications
│       └── kustomization.yaml
├── age.agekey
├── LICENSE
└── README.md
```

### Directory Responsibilities

| Directory | Description |
|-----------|-------------|
| `infra/talos/` | Talos Linux configuration (applied outside cluster) |
| `k8s/flux/` | Flux system components (auto-managed) |
| `k8s/infrastructure/` | Cluster infrastructure: CNI, ingress, cert-manager, monitoring |
| `k8s/apps/` | User applications deployed on the cluster |

## Bootstrap Flux

### 1. Run Bootstrap Command

```bash
flux bootstrap github \
  --token-auth \
  --owner=aoshimash \
  --repository=homelab-k8s \
  --branch=main \
  --path=k8s/flux \
  --personal
```

**Parameters explained:**

| Parameter | Value | Description |
|-----------|-------|-------------|
| `--token-auth` | - | Use GitHub PAT for authentication (HTTPS) |
| `--owner` | `aoshimash` | GitHub username or organization |
| `--repository` | `homelab-k8s` | Repository name |
| `--branch` | `main` | Git branch to use |
| `--path` | `k8s/flux` | Path in repository for Flux manifests |
| `--personal` | - | Repository is owned by a personal account |

### 2. Verify Installation

Check that Flux components are running:

```bash
# Check Flux pods
kubectl -n flux-system get pods

# Check Flux components status
flux check

# View GitRepository status
flux get sources git

# View Kustomization status
flux get kustomizations
```

Expected output:
- All pods in `flux-system` namespace are `Running`
- GitRepository shows `Ready` with latest commit
- Kustomization shows `Ready` and `Applied`

## Post-Bootstrap Setup

### 1. Create Infrastructure Kustomization

Create the directory structure for infrastructure components:

```bash
mkdir -p k8s/infrastructure/controllers
mkdir -p k8s/infrastructure/configs
```

Create `k8s/infrastructure/kustomization.yaml`:

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources: []
  # - controllers/ingress-nginx
  # - controllers/cert-manager
  # - configs/cluster-issuers
```

Create the Flux Kustomization to deploy infrastructure:

```yaml
# k8s/flux/infrastructure.yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: infrastructure
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./k8s/infrastructure
  prune: true
  wait: true
```

### 2. Create Apps Kustomization

Create the directory structure for applications:

```bash
mkdir -p k8s/apps
```

Create `k8s/apps/kustomization.yaml`:

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources: []
  # - my-app
```

Create the Flux Kustomization to deploy apps (depends on infrastructure):

```yaml
# k8s/flux/apps.yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: apps
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./k8s/apps
  prune: true
  wait: true
  dependsOn:
    - name: infrastructure
```

### 3. Commit and Push

```bash
git add k8s/
git commit -m "feat: add infrastructure and apps kustomizations"
git push
```

### 4. Verify Reconciliation

```bash
# Force reconciliation
flux reconcile source git flux-system

# Check all kustomizations
flux get kustomizations

# Watch for changes
flux get kustomizations --watch
```

## SOPS Integration (Optional)

To manage secrets with SOPS and Age encryption:

### 1. Create SOPS Secret

```bash
kubectl create secret generic sops-age \
  --namespace=flux-system \
  --from-file=age.agekey=./age.agekey
```

### 2. Configure Flux Kustomization for SOPS

Add decryption configuration to your Kustomizations:

```yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: infrastructure
  namespace: flux-system
spec:
  # ... other fields ...
  decryption:
    provider: sops
    secretRef:
      name: sops-age
```

## Useful Commands

### Reconciliation

```bash
# Reconcile GitRepository
flux reconcile source git flux-system

# Reconcile all Kustomizations
flux reconcile kustomization flux-system
flux reconcile kustomization infrastructure
flux reconcile kustomization apps
```

### Debugging

```bash
# View Flux logs
flux logs

# View specific controller logs
flux logs --kind=Kustomization --name=flux-system

# Check events
kubectl -n flux-system get events --sort-by='.lastTimestamp'
```

### Suspend/Resume

```bash
# Suspend reconciliation (for maintenance)
flux suspend kustomization flux-system

# Resume reconciliation
flux resume kustomization flux-system
```

## Troubleshooting

### GitRepository Not Ready

```bash
# Check GitRepository status
flux get sources git -A

# Describe for detailed error
kubectl -n flux-system describe gitrepository flux-system
```

Common causes:
- Invalid GitHub token
- Repository not accessible
- Branch doesn't exist

### Kustomization Failed

```bash
# Check Kustomization status
flux get kustomizations -A

# View detailed error
kubectl -n flux-system describe kustomization <name>

# Check logs
flux logs --kind=Kustomization --name=<name>
```

Common causes:
- Invalid YAML syntax
- Missing resources referenced in kustomization.yaml
- RBAC permissions

### Force Sync

```bash
# Delete and recreate the source
flux reconcile source git flux-system --force
```

## Uninstall Flux

To completely remove Flux from the cluster:

```bash
flux uninstall --silent
```

> **Warning**: This will remove all Flux components and CRDs. Resources deployed by Flux will remain but won't be managed anymore.
